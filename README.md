# PM3 WebUI 一键工具

一个面向 **Proxmark3 / PM3（ICEMAN固件）** 的本地 Web 操作面板。

本项目不实现任何 RFID / NFC 底层功能，也不修改 Proxmark3 固件或客户端逻辑。它只是把官方 / Iceman 版 `pm3` 命令行客户端的常用命令封装成网页按钮、表单和实时控制台，方便在浏览器中完成日常操作。

---

## 项目定位

### 本工具是什么

* 一个将PM3命令封装成webUI，用于一键读取非接触式卡片的工具；

### 本工具不是什么

* 不是 Proxmark3 固件；
* 不是 Proxmark3 客户端的二次开发；
* 不包含 Proxmark3 固件或客户端源码；
* 不实现任何独立的 RFID / NFC 算法；
* 不绕过、不修改、不替代 Iceman 客户端本身的能力边界。

所有实际功能，例如读卡、识别、密钥检测、dump、restore、模拟、脚本调用等，均来自外部的 Proxmark3 客户端。本工具只负责把命令发送给客户端，并把执行结果展示出来。

## 快速开始

### 1. 准备 Proxmark3 客户端

请先按 Iceman 官方文档编译客户端，确保本机已经存在可执行的客户端文件，例如：

* macOS / Linux：`pm3` 或 `proxmark3`
* Windows：`proxmark3.exe`

并确认命令行中可以正常连接设备。

---

### 2. 启动 WebUI

根据系统选择对应启动方式：

| 系统      | 启动方式                     |
| ------- | ------------------------ |
| macOS   | 双击 `macos_start.command` |
| Linux   | 终端执行 `./linux_start.sh`  |
| Windows | 双击 `windows_start.bat`   |

也可以直接在终端中运行：

```bash
python3 pm3_web.py
```

Windows 可使用：

```bat
py -3 pm3_web.py
```

启动后程序会自动打开浏览器，访问类似下面的地址：

```text
http://127.0.0.1:<随机端口>/
```

## 首次使用配置

打开页面后，请先配置左上角区域。

### 1. 指定 pm3 客户端路径

左上角第一个输入框需要填写 Proxmark3 客户端的完整路径。

也就是你本地 proxmark3 项目中的：

* `pm3` 启动脚本；
* `proxmark3` 可执行文件；
* Windows 下的 `proxmark3.exe`。

示例：

```text
/Users/windowsnoeditor/Desktop/proxmark3/pm3
```

macOS / Linux 通常类似：

```text
/你的路径/proxmark3/pm3
```

Windows 通常类似：

```text
D:\proxmark3\client\proxmark3.exe
```

程序会尝试自动填入常见路径，但如果你的 proxmark3 项目不在工具目录旁边，请手动修改为实际路径。

---

### 2. 选择串口

点击“扫描”按钮，选择 Proxmark3 设备对应的串口。

常见示例：

| 系统      | 串口示例                          |
| ------- | ----------------------------- |
| macOS   | `/dev/tty.usbmodemiceman1`    |
| Linux   | `/dev/ttyACM0`、`/dev/ttyUSB0` |
| Windows | `COM3`、`COM4`                 |

---

### 3. 设置保存目录

设置 dump、key、log 等文件的保存目录。

建议使用一个独立目录，方便后续整理数据文件。

---

### 4. 点击连接

确认 pm3 路径、串口、保存目录无误后，点击“连接”。

连接成功后，即可通过页面按钮或右侧控制台执行命令。

## 平台支持

| 系统      | 后端方式      | 支持情况  | 说明                      |
| ------- | --------- | ----- | ----------------------- |
| macOS   | `pty` 伪终端 | 已实测   | 输出实时流式显示，体验最佳           |
| Linux   | `pty` 伪终端 | 理论支持  | 原理与 macOS 一致            |
| Windows | pipe 管道   | 实验性支持 | 长任务期间可能不会实时滚动，命令完成后整段返回 |

### Windows 注意事项

Windows 标准库没有与 Unix `pty` 完全一致的伪终端能力，因此当前采用 pipe 方式调用客户端。

这意味着：

* 短命令通常可以正常执行；
* 长任务可能不会实时刷新输出；
* 执行结束后才会一次性显示结果；
* 需要使用 `proxmark3.exe`；
* 串口需要选择 `COM` 口，例如 `COM3`。

如果希望在 Windows 上获得更接近 macOS / Linux 的实时体验，建议在 WSL2 中运行本工具，并按 Linux 环境连接设备。

## 功能模块

### 设备与识别

提供常用设备检查和卡片识别命令入口，例如：

* `hw version`
* `hw status`
* `hw tune`
* `auto`
* `hf search`
* `lf search`
* `hf 14a info`

适合用于确认设备状态、天线状态、卡片类型和基础信息。

---

### MIFARE Classic 辅助操作

针对 MIFARE Classic 常用流程进行了按钮化封装，包括：

* 智能获取密钥；
* `fchk` 检查；
* `autopwn`；
* dump；
* restore；
* key 文件管理；
* 扇区数据展示。

其中“智能获取密钥”会根据客户端输出和卡片特征，自动选择合适的 Proxmark3 客户端命令流程。实际能力完全取决于当前 Iceman 客户端版本和设备支持情况。

---

### 脚本调用

支持调用 Proxmark3 客户端内置的部分 Python 脚本，例如：

* `fm11rf08s_recovery`
* `fm11rf08s_full -r`
* `mf_backdoor_dump`
* `ntag22x_suncmac_recovery`
* `hf_mfu_uscuid`
* `script list`

这些脚本来自 Proxmark3 客户端，本工具只负责调用和展示结果。

---

### 复制卡片辅助流程

提供源卡读取、新卡写入、UID 校验等流程化按钮，降低重复输入命令的成本。

支持的流程取决于卡片类型、目标卡类型和 Proxmark3 客户端能力，例如：

* Gen1a 卡片写入；
* Gen2 卡片写入；
* dump 后 restore；
* UID 检查；
* 写入后验证。

请仅在本人拥有或已明确授权的卡片上使用。

---

### 手机复制辅助流程

提供向导式流程，用于辅助以下操作：

1. 读取源卡信息；
2. 使用 PM3 模拟卡片；
3. 使用手机复制 UID；
4. 验证手机端 UID 是否一致；
5. 根据需要继续执行数据写入或验证流程。

该流程只是对命令步骤进行整理，不保证所有手机、系统版本或卡片类型均可成功。

---

### 魔术卡 / Ultralight / 低频

提供部分常用命令入口，包括：

* Gen1a / Gen2 相关操作；
* `cload`
* `csetuid`
* `cwipe`
* `hf mfu`
* `lf em410x`
* `lf t55xx`
* `lf hid`

具体支持情况以当前 Proxmark3 客户端输出为准。

---

### 状态面板

页面左下角提供状态展示区域，包括：

* 当前卡片信息；
* 已恢复密钥；
* 扇区 KeyA / KeyB；
* 扇区块数据；
* 尾块高亮显示；
* 当前任务执行状态。

该面板主要用于整理 Proxmark3 输出，方便快速查看结果。

---

### 控制台

页面右侧为 Proxmark3 控制台输出区。

支持：

* 实时显示 pm3 输出；
* 查看命令执行过程；
* 手动输入任意 pm3 命令；
* 查看错误、提示、脚本输出等信息。

macOS / Linux 下输出为实时流式显示。

Windows 下可能需要等待命令执行结束后一次性返回。

## 常见问题

### 找不到 pm3 客户端

请检查左上角的客户端路径是否正确。

macOS / Linux 下通常是：

```text
/你的路径/proxmark3/pm3
```

Windows 下通常是：

```text
D:\proxmark3\client\proxmark3.exe
```

路径中如果包含空格，建议优先使用无空格路径，或确认系统能正确识别该路径。

---

### 连接不上设备

可以按以下顺序排查：

1. 确认 Proxmark3 已通过 USB 连接；
2. 确认串口选择正确；
3. 确认没有其他程序正在占用串口；
4. 确认命令行中直接运行 `pm3` 可以连接；
5. macOS / Linux 下检查串口权限；
6. Windows 下确认选择的是正确的 `COM` 口。

---

### 页面打开了，但命令没有输出

请检查：

1. pm3 客户端路径是否正确；
2. 客户端文件是否有执行权限；
3. 串口是否选择正确；
4. Proxmark3 客户端是否能在终端中单独运行；
5. Windows 下是否正在执行长任务，需要等待命令结束后返回。

---

### Windows 为什么不是实时输出

Windows 当前使用 pipe 管道方式调用客户端，无法像 macOS / Linux 的 `pty` 一样完整模拟交互式终端。

因此 Windows 下部分长任务可能不会实时刷新输出，而是在命令执行完成后一次性显示。
